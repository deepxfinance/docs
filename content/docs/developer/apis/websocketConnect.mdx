          Real-time data streaming via WebSocket.


          **Endpoint**

          - URL: `/v2/ws` (GET, upgrade to WebSocket)

          - Protocol: JSON messages, utf-8


          **Markets**

          - Spot: `{"type":"spot","name":"ETH/USDC"}` or
          `{"type":"spot","pair":"0x..."}` (one of name/pair required)

          - All (aggregate): `{"type":"all"}` (supports `volume_stats` plus user
          portfolio channels)

          - Lending (aggregate): `{"type":"lending"}` (supports `total_deposits`
          and `total_borrows`)


          **Subscriptions**

          - Simple channels: `orderbook`, `trades`, `price_change_1h`,
          `price_change_24h`, `volume_stats`, `latest_price`, `oracle_price`,
          `mark_price`, `funding_rate`, `avg_funding_rate_24h`,
          `funding_history`, `open_interest`, `long_short_ratio`,
          `total_deposits`, `total_borrows`

          - Lending pool status: `lending_pool_status` (lending market only)

          - Lending pool status by asset (case-insensitive):
            - `{"channel":"lending_pool_status","asset":"USDC"}`
          - Asset-filtered totals (lending only, case-insensitive asset):
            - `{"channel":"total_deposits","asset":"USDC"}`
            - `{"channel":"total_borrows","asset":"usdc"}`
          - User perp positions (perp only, address required):
            - `{"channel":"user_perp_positions","address":"0x...","status":"open|closed|all"}`
            - Legacy filter: `{"channel":"user_perp_positions","address":"0x...","onlyClosed":true}` (same as `status:"closed"`)
            - Wallet scope (expand to all subaccounts): `{"channel":"user_perp_positions","address":"0x...","addressType":"wallet"}`
          - User perp open orders (perp only, address required):
            - `{"channel":"user_perp_open_orders","address":"0x..."}`
            - Wallet scope (expand to all subaccounts): `{"channel":"user_perp_open_orders","address":"0x...","addressType":"wallet"}`
          - User perp order history (perp only, address required):
            - `{"channel":"user_perp_order_history","address":"0x..."}`
          - User spot open orders (spot only, address required):
            - `{"channel":"user_spot_open_orders","address":"0x..."}`
            - Wallet scope (expand to all subaccounts): `{"channel":"user_spot_open_orders","address":"0x...","addressType":"wallet"}`
          - User spot order history (spot only, address required):
            - `{"channel":"user_spot_order_history","address":"0x..."}`
          - User perp trade history (perp only, address required):
            - `{"channel":"user_perp_trade_history","address":"0x..."}`
          - User spot trade history (spot only, address required):
            - `{"channel":"user_spot_trade_history","address":"0x..."}`
          - User incremental channels push behavior:
            - `user_perp_order_history`, `user_spot_open_orders`: incremental only (no snapshot/compensation)

            - Use REST endpoints for full snapshot/history
          - User balances (all only, address required):
            - `{"channel":"user_balances","address":"0x..."}`
          - User total deposits/borrows (all only, address required):
            - `{"channel":"user_total_deposits","address":"0x..."}`
          - User unified margin ratio (all only, address required):
            - `{"channel":"user_portfolio_margin_ratio","address":"0x..."}`
          - User portfolio push interval:
            - batched every ~1s (only pushes on change; force push every ~30s)
          - Candles channel: `{"time_frame":
          "1m|3m|5m|15m|30m|1h|2h|4h|8h|12h|1d|3d|1w|1M"}`

          **Server → Client**

          - Ack:
          `{"type":"subscribed","market":{...},"subscriptions":[...],"message":"ok"}`

          - Ack (subs changed):
          `{"type":"subscriptions_changed","market":{...},"subscriptions":[...],"message":"ok"}`

          - Ack (options changed):
          `{"type":"options_changed","options":{...},"message":"ok"}`

          - Data push:
          `{"type":"data","channel":"orderbook","market":{...},"data":{...},"timestamp":1700000000}`
          (perp market includes `id`)

          - Data push (lending totals):

          ```json

          {
            "type": "data",
            "channel": "total_deposits",
            "market": {"type": "lending"},
            "data": {
              "total": 234.56,
              "items": [
                {"market_id": 1, "asset": "USDC", "value": 10.0},
                {"market_id": 2, "asset": "USDC", "value": 5.0}
              ]
            },
            "timestamp": 1700000000
          }

          ```

          - Data push (lending pool status):

          ```json

          {
            "type": "data",
            "channel": "lending_pool_status",
            "market": {"type": "lending"},
            "data": {
              "items": [
                {
                  "market_id": 1,
                  "asset": "USDC",
                  "total_supplied": 123.45,
                  "supply_cap": 100.0,
                  "supply_apr": 0.06,
                  "initial_asset_weight": 0.5,
                  "initial_borrow_weight": 0.7,
                  "maintenance_asset_weight": 0.4,
                  "maintenance_borrow_weight": 0.3,
                  "total_borrowed": 12.34,
                  "borrow_cap": 10.0,
                  "borrow_apr": 0.08
                }
              ]
            },
            "timestamp": 1700000000
          }

          ```

          - Error:
          `{"type":"error","code":"INVALID_SUBSCRIPTION","message":"..." }`

          - Heartbeat: `{"type":"pong","timestamp":1700000000}`


          **Notes**

          - One connection can manage multiple subscriptions for the selected
          market.

          - Use `change_market` to switch market without reconnecting.

          - Use `change_subscriptions` / `change_options` to update settings
          without reconnecting.

          - Server replies to WebSocket-level ping/pong frames and to `ping`
          action with `pong`.

- Ack (options changed): `{"type":"options_changed","options":{...},"message":"ok"}`
- Data push: `{"type":"data","channel":"orderbook","market":{...},"data":{...},"timestamp":1700000000}` (perp market includes `id`)

**Notes**
- One connection can manage multiple subscriptions for the selected market.
- Use `change_market` to switch market without reconnecting.
- Use `change_subscriptions` / `change_options` to update settings without reconnecting.
- Server replies to WebSocket-level ping/pong frames and to `ping` action with `pong`.
