---
title: Funding Rates
---

Funding Rate is a periodic rate applied to perpetual positions to calculate the funding payment exchanged between long and short positions. This mechanism ensures that the price of the perpetual contract stays close to the underlying asset's index price.

- **Positive Funding Rate**: Longs pay shorts. This occurs when the perpetual contract price is higher than the index price.
- **Negative Funding Rate**: Shorts pay longs. This occurs when the perpetual contract price is lower than the index price.

## Funding Rate Calculation

The funding rate is calculated based on the **Premium Index** and a fixed **Base Interest Rate**. The Base Interest Rate ($R_\text{base}$) is set to **0.01% per 8 hours**.

### Premium Index

The premium index reflects the price difference between the perpetual contract and the index price. It is calculated as:

```math
\text{PremiumIndex} = \frac{\text{Max}(0, \text{ImpactBidPrice} - \text{IndexPrice}) - \text{Max}(0, \text{IndexPrice} - \text{ImpactAskPrice})}{\text{IndexPrice}}
```

where:
- **Impact Bid Price**: The average execution price to trade the Impact Notional Value on the bid side of the order book.
- **Impact Ask Price**: The average execution price to trade the Impact Notional Value on the ask side of the order book.

**Impact Notional Value** is calculated by dividing an **Impact Margin Value** (set at $100) by the [Initial Margin Ratio](/glossary#initial-margin-ratio-imr) for each market. This value determines the size of the hypothetical trade used to calculate impact prices, providing a standardized measure of how the perpetual contract price deviates from the index price under realistic trading conditions. 

### Average Premium Index

The premium index is sampled periodically (currently every 5 seconds). The Average Premium Index ($P$) is the average of the premium index over the funding interval (currently every hour).

### Funding Rate Formula

The funding rate is calculated by taking the 8-hour base interest rate and adding a clamped premium component.

```math
\text{FundingRate} = P + \text{clamp}(R_\text{base} - P, \text{lowerBound}, \text{upperBound})
```

The **clamp function** restricts a value to a specific range [lowerBound, upperBound], where the lower bound and upper bound are set to -0.02% and 0.02% respectively. 

If the input value is below the minimum, it returns the minimum; if above the maximum, it returns the maximum; otherwise, it returns the input value unchanged. This ensures the funding rate is equal to the base interest rate when the difference between the base interest rate and the average premium index is within the clamp range.

<Callout>
The funding rate formula applies to an 8-hour period. However, funding payments occur every hour at one-eighth of the computed 8-hour rate.
</Callout>

### Funding Rate Limits

To protect traders from extreme market conditions, the funding rate is subject to upper and lower limits:

- **Maximum Funding Rate (Cap)**: +0.2% per hour
- **Minimum Funding Rate (Floor)**: -0.2% per hour

These limits ensure that funding payments remain within reasonable bounds, even during periods of significant market volatility or imbalance between long and short positions. If the calculated funding rate exceeds these limits, it will be capped at the maximum or minimum value.

## Funding Interval and Payment

Funding payments are exchanged every hour.

The amount paid or received is calculated as the product of the [Notional Value](/glossary#notional-value) and the funding rate.

```math
\text{FundingAmount} = \text{NotionalValue} \times \text{FundingRate}
```

## Example

### Example 1: Funding Rate Equals Base Interest Rate

**Parameters:**
- **Base Interest Rate (I)**: 0.01%
- **Average Premium Index (P)**: 0.04%

**Step 1: Calculate the difference**
```math
I - P = 0.01\% - 0.04\% = -0.03\%
```

**Step 2: Clamp the difference**
The clamp range is $[-0.05\%, 0.05\%]$. If the difference is within this range, it is clamped to the range.
```math
\text{clamp}(I - P, -0.05\%, 0.05\%) = I - P
```

**Step 3: Final Funding Rate**
```math
F = P + \text{clamp}(I - P, -0.05\%, 0.05\%) = P + (I - P) = I = 0.01\%
```

In this case, the Funding Rate equals the Base Interest Rate because the premium was within the clamp range.

### Example 2: Funding Rate Below Base Interest Rate

**Parameters:**
- **Base Interest Rate (I)**: 0.01%
- **Average Premium Index (P)**: -0.06%

**Step 1: Calculate the difference**
```math
I - P = 0.01\% - (-0.06\%) = 0.07\%
```

**Step 2: Clamp the difference**
The clamp range is $[-0.05\%, 0.05\%]$. If the difference is above the maximum, it is clamped to the maximum.
```math
\text{clamp}(I - P, -0.05\%, 0.05\%) = 0.05\%
```

**Step 3: Final Funding Rate**
```math
F = P + \text{clamp}(I - P, -0.05\%, 0.05\%) = -0.06\% + 0.05\% = -0.01\%
```
